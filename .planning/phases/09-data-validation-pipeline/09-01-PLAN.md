# Phase 9 Plan: Data Validation Pipeline

## Goal
Implement a pre-generation data validation pipeline that cross-checks key financial metrics (revenue, EPS, margins, capex, FCF) for consistency and plausibility, distinguishes GAAP vs non-GAAP figures, and surfaces validation results to the report generation narrative.

## Must Haves
- [ ] VALID-01: Metrics are extracted and cross-checked before narrative generation
- [ ] VALID-02: Deviations beyond configurable thresholds trigger warnings or blocks
- [ ] VALID-03: GAAP and non-GAAP versions of EPS are distinguished when present
- [ ] VALID-04: Validation results influence narrative confidence levels via prompt injection
- [ ] All existing functionality preserved
- [ ] Python syntax valid on all modified files

## Tasks

### Task 1: Create DataValidationService module
**read_first:**
  - backend/app/utils/language_integrity.py (validation result pattern)
  - backend/app/services/zep_tools.py (simulation context structure)
**action:**
  Create `backend/app/services/data_validation.py` with:
  - `MetricEntry` dataclass: name, value, unit, period, source, metric_type
  - `ValidationReport` dataclass: metrics, discrepancies, confidence_level, gaap_non_gaap_notes, is_valid
  - `DataValidationService` class:
    - `extract_metrics(context: dict) -> List[MetricEntry]` — regex-based extraction from simulation context facts and requirement text
    - `validate_structural(metrics: List[MetricEntry]) -> List[dict]` — Tier 1 checks (margin ordering, positive revenue, capex < revenue)
    - `validate_against_reference(metrics: List[MetricEntry], reference_facts: List[str]) -> List[dict]` — Tier 2 checks (deviation from reference ranges)
    - `check_gaap_non_gaap(metrics: List[MetricEntry]) -> dict` — detect GAAP vs non-GAAP EPS presence
    - `validate(simulation_requirement: str, context: dict, thresholds: dict) -> ValidationReport` — orchestrate all checks
  - Default thresholds dict for economia profile
  - Portuguese comments and variable names where appropriate
**acceptance_criteria:**
  - `python -m py_compile backend/app/services/data_validation.py` passes
  - Module is importable without errors
  - Unit test stub runs successfully (test_validate_basic.py)

### Task 3: Hook validation into ReportAgent.generate_report()
