# Phase 11 Plan: Neutrality and Bias Audit

## Goal
Implement post-generation bias detection and neutrality enforcement for economy reports. Detect confirmation bias, ensure bull/bear balance, calibrate claim strength to evidence quality, and enforce quantified competitive analysis.

## Must Haves
- [ ] NEUT-01: Bias score computed; reports leaning consistently bullish/bearish are flagged
- [ ] NEUT-02: Bull and bear cases have comparable depth and specificity
- [ ] NEUT-03: Strong claims require strong evidence; speculative claims use conditional language
- [ ] NEUT-04: Competitive analysis quantifies metrics where data exists
- [ ] All existing functionality preserved
- [ ] Python syntax valid on all modified files

## Tasks

### Task 1: Create BiasAuditService module
**read_first:**
  - backend/app/services/data_validation.py (audit report pattern)
  - backend/app/utils/language_integrity.py (pattern-based validation)
**action:**
  Create `backend/app/services/bias_audit.py` with:
  - `BiasDimension` enum: SENTIMENT_BALANCE, CLAIM_CALIBRATION, COMPETITIVE_QUANTIFICATION
  - `BiasReport` dataclass: bias_score (0.0-1.0), dimensions (dict), warnings (list), is_balanced (bool)
  - Keyword dictionaries:
    - BULLISH_KEYWORDS: crescimento, recuperacao, oportunidade, expansao, alta, superar, forte, robusto, otimista, upside, rally, boom, melhora, avanco, ganho
    - BEARISH_KEYWORDS: queda, risco, preocupacao, contracao, baixa, fracasso, fragil, vulneravel, pessimista, downside, crash, piora, recuo, perda, estresse
    - STRONG_CLAIM_MARKERS: certamente, inevitavelmente, sem duvida, com certeza, absolutamente, inquestionavelmente, definitivamente, ira (futuro assertivo), sera
    - CONDITIONAL_MARKERS: pode, talvez, caso, se, provavelmente, possivelmente, em torno de, aproximadamente, estimado, projetado
  - `BiasAuditService` class:
    - `audit_sections(section_contents: List[str]) -> BiasReport`
    - `_analyze_sentiment_balance(texts) -> dict` — counts bullish/bearish keywords, computes ratio
    - `_analyze_claim_calibration(texts) -> dict` — finds strong claim markers without 📊 (fato) tags; checks for conditional markers with 🔮 (hipotese) tags
    - `_analyze_competitive_quantification(texts) -> dict` — detects competitive analysis paragraphs lacking numeric patterns
    - `_compute_composite_score(dimensions) -> float` — weighted composite (sentiment 40%, claim 35%, competitive 25%)
  - Thresholds: warning if score < 0.80, critical if < 0.60
**acceptance_criteria:**
  - `python -m py_compile backend/app/services/bias_audit.py` passes
  - Module is importable and testable with mock text
