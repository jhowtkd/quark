# Plan 08-01: Data Provenance and Labeling Implementation

## Goal
Implement defense-in-depth provenance labeling so every quantitative claim in economia-profile reports carries an explicit source-type emoji tag (📊 Fato, 🔮 Hipótese, ⚠️ Dados insuficientes), includes a Data Sources section, and logs provenance metadata for observability.

## must_haves
- Every quantitative claim in generated reports is tagged with 📊, 🔮, or ⚠️.
- Reports include a "Fontes de Dados" section at the end.
- Simulated/projected data never appears as verified fact.
- Provenance metadata is logged in `agent_log.jsonl`.
- Each `meta.json` carries `provenance_version: "1.0"`.
- Old reports (pre-phase-8) display as-is without modification.

    1. Create a new method `_validate_provenance_tags()` in `ReportAgent` (near `_post_process_report`):
       ```python
       @classmethod
       def _validate_provenance_tags(cls, content: str) -> tuple[bool, list[str]]:
           """
           Validate that quantitative claims have provenance emoji tags.
           Returns (is_valid, warnings).
           """
           import re
           warnings = []
           # Pattern: numbers/money/percentages not followed by provenance emoji within same sentence
           # Look for numeric patterns: $X, X%, X bilhoes, X milhoes, dates like 2025, Q1 2026
           numeric_pattern = re.compile(
               r'(?:US\$\s*[\d,.]+|\d+[,.]?\d*\s*(?:%|bilh(?:o|õ)es|milh(?:o|õ)es|mil)|\b20\d{2}\b|Q[1-4]\s+20\d{2})',
               re.IGNORECASE
           )
           # Split into sentences roughly
           sentences = re.split(r'[.!?
]+', content)
           for sentence in sentences:
               if numeric_pattern.search(sentence) and not re.search(r'[📊🔮⚠️]', sentence):
                   # Skip table rows and headers
                   if sentence.strip().startswith('|') or sentence.strip().startswith('#'):
                       continue
                   snippet = sentence.strip()[:80]
                   warnings.append(f"Untagged numeric claim: {snippet}...")
           is_valid = len(warnings) == 0
           return is_valid, warnings
       ```

    2. In `assemble_full_report()` (line 2746), after calling `_post_process_report()`, add:
       ```python
       # Validate provenance tags
       is_valid, warnings = cls._validate_provenance_tags(md_content)
       if not is_valid:
           logger.warning(f"[Provenance] Report {report_id} has {len(warnings)} untagged claims")
           for w in warnings[:10]:  # Log first 10
               logger.warning(f"[Provenance] {w}")
           # Write warnings to a sidecar file for debugging
           warn_path = os.path.join(folder, 'provenance_warnings.txt')
           with open(warn_path, 'w', encoding='utf-8') as f:
               f.write(f"# Provenance Warnings for {report_id}\n\n")
               for w in warnings:
                   f.write(f"- {w}\n")
       ```

    3. If `agent.report_logger` exists, log the validation result:

           agent.report_logger.log(
               action="provenance_validation",
               stage="generating",
               details={
                   "is_valid": is_valid,
                   "warning_count": len(warnings),
                   "warnings": warnings[:20]
               }
           )

       (Note: since `assemble_full_report` is a classmethod, access the logger via the report_id or skip if no instance available.)

  <read_first>

  <action>
    In `backend/app/api/report.py`, in the `generate_report()` function:

    1. After creating the `ReportAgent` instance in `run_generate()` (around line 146), add a version check:
       ```python
       # Provenance version check — abort if mismatch
       if getattr(agent, 'require_provenance', False):
           # This is a provenance-aware agent; verify consistency
           pass  # Normal path
       else:
           # Agent does not support provenance but profile expects it
           if profile.require_provenance and profile.profile_type == UserProfile.ECONOMIA:
               raise RuntimeError(
                   "Provenance version mismatch: simulation started before provenance support was deployed. "
                   "Please restart the simulation."
               )
       ```

    2. Actually, a simpler and safer approach: In `ReportAgent.__init__()`, add:
       ```python
       self.provenance_version = "1.0"
       ```
       Then in `generate_report()`, before starting generation, check if the profile requires provenance but the agent was initialized from a stale state. Since the abort-on-deployment concern is about in-flight reports during code deployment, the most pragmatic fix is to store the provenance capability version at task creation time in the TaskManager metadata, and in `run_generate()` compare it against the current agent's capability.

    3. **Simplified approach:** Add `provenance_version: "1.0"` to the task metadata when creating the task (line 121-129):
       ```python
       task_id = task_manager.create_task(
           task_type="report_generate",
           metadata={
               "simulation_id": simulation_id,
               "graph_id": graph_id,
               "report_id": report_id,
               "profile": profile.profile_type.value,
               "provenance_version": "1.0" if profile.require_provenance else None
           }
       )
       ```
       And in `run_generate()`, if `profile.require_provenance` is True but the agent does not have the `require_provenance` attribute (indicating stale code), fail the task with a clear message.

    The key requirement from D-15 is: no hybrid/partial-provenance reports. If the code deploys mid-generation, the task will complete with the old code (no provenance) — which is actually fine because the report will have `provenance_version` missing in meta.json, and old reports are displayed as-is (D-12). The abort is only needed if we actively detect a mismatch. Given the threading model, the simplest implementation is:
    - The new code always runs provenance when `profile.require_provenance` is True.
    - Old in-flight reports use old code and finish without provenance.
    - This satisfies D-12 (old reports as-is) without needing explicit abort logic.

    **Revised action:** Document this behavior in code comments and add a defensive check in `ReportAgent.generate_report()`:
    ```python
    # Defensive: ensure we don't generate partial-provenance reports
    if getattr(self, 'require_provenance', False) and not hasattr(self, 'provenance_version'):
        raise RuntimeError("Provenance support incomplete: agent missing provenance_version")
    ```
  </action>
