# Phase 2 — UI Design Contract

> Visual and interaction contract for Component Refactoring. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Scope: refactor in-place CSS for buttons, inputs, and cards across all `.vue` files to match Blueprint Noir analog brutalism aesthetic.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Vue 3 SFC in-place styles) |
| Icon library | none (uses Unicode glyphs ◇ ◈ ◆) |
| Font | Space Grotesk + Work Sans |

**Notes:**
- Global CSS variables already established in `App.vue` during Phase 1 (`--color-primary`, `--color-background`, `--color-surface-container-low`, `--color-on-background`, `--font-machine`, `--font-human`).
- No component extraction; all changes are in-place within `<style>` blocks per D-01.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: 12px compact padding inside small icon buttons (existing `.action-btn` in Process.vue); 14px vertical padding on `.form-input` (converts to 16px under this contract).

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 | 1.5 | Work Sans |
| Label | 14px | 600 | 1.4 | Space Grotesk |
| Heading | 20px | 600 | 1.2 | Space Grotesk |
| Display | 28px | 600 | 1.15 | Space Grotesk |

**Notes:**
- All buttons use `font-family: var(--font-machine)` (Space Grotesk) at 14px–16px, weight 600.
- Inputs use Work Sans 16px at weight 400.
- Card titles use Space Grotesk 16px–20px, weight 600.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #f9f9f9 | Page background, canvas surfaces |
| Secondary (30%) | #e2e2e2 | Card backgrounds, elevated containers (`--color-surface-container-highest`) |
| Accent (10%) | #000000 | Primary action fills, borders, focus rings, block shadows (`--color-primary`) |
| Destructive | #ba1a1a | Error states, destructive actions only (`--color-error`) |

Accent reserved for:
- Primary action button backgrounds and text
- Bottom borders on focused inputs
- Solid block shadows on hover (`box-shadow: 4px 4px 0 #000000`)
- Active/focus outlines (2px solid `#000000`)

**Tonal layering rule (from UI-04):**
- Cards must sit on `#e2e2e2` against the `#f9f9f9` page background.
- No drop shadows for elevation; depth is achieved exclusively by tonal shifts and the hover block shadow.

## Component Contracts

### Buttons

Applies to: `.login-button`, `.action-btn`, `.start-engine-btn`, `.next-step-btn`, `.modal-btn`, `.send-btn`, `.close-btn`

| Property | Contract |
|----------|----------|
| Border radius | 0px |
| Border | 2px solid `#000000` for secondary/outline variants; none for primary fill |
| Background (primary) | `#000000` |
| Color (primary) | `#e2e2e2` |
| Background (secondary) | `#f9f9f9` |
| Color (secondary) | `#000000` |
| Padding | 14px 24px (large), 12px 16px (compact) |
| Font | Space Grotesk, 14px–16px, weight 600 |
| Hover (primary) | Invert to `background: #f9f9f9; color: #000000; box-shadow: 4px 4px 0 #000000;` |
| Hover (secondary) | `background: #000000; color: #e2e2e2; box-shadow: 4px 4px 0 #000000;` |
| Focus | `outline: 2px solid #000000; outline-offset: 2px;` |
| Disabled | `opacity: 0.4; cursor: not-allowed; box-shadow: none;` |

### Inputs

Applies to: `.form-input`, `.chat-input`, `.survey-input`, `.code-input`

| Property | Contract |
|----------|----------|
| Border radius | 0px |
| Background | transparent |
| Border | none except `border-bottom: 2px solid #777777` (`--color-outline`) |
| Padding | 14px 0px (bottom-border only) or 16px (when inside a filled container) |
| Font | Work Sans, 16px, weight 400 |
| Focus | `border-bottom-color: #000000; outline: none;` |
| Error | `border-bottom-color: #ba1a1a;` |
| Disabled | `opacity: 0.5; cursor: not-allowed;` |

**Filled container variant (e.g. `.code-input` wrapper):**
- Wrapper gets `background: #f3f3f3` (`--color-surface-container-low`) and `border-radius: 0px`.
- Input inside remains transparent with no independent border.

### Cards

Applies to: `.project-card`, `.step-card`, `.info-card`, modal panels, graph panels

| Property | Contract |
|----------|----------|
| Border radius | 0px |
| Background | `#e2e2e2` (`--color-surface-container-highest`) |
| Border | none (use tonal shift against `#f9f9f9` background) |
| Padding | 16px–24px |
| Hover | `box-shadow: 4px 4px 0 #000000; transform: translate(-2px, -2px);` |

**History database cards note:**
- Remove existing colored status dots (green/orange/yellow) and replace with monochrome tones (`#000000`, `#5e5e5e`, `#c6c6c6`).
- Remove card-bottom-line gradient animations; replace with static 2px `#000000` bottom border if needed.
