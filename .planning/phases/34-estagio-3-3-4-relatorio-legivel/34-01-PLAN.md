# 34-01 — Modo Foco no Relatorio

## Goal
Permitir ao usuario ler o relatorio como um documento final, sem a interface tecnica de workflow (timeline de agentes, logs de console, tool calls). O modo foco expande o painel de conteudo do relatorio, oculta o painel direito de workflow e o console inferior, e mantem o indice de secoes navegavel.

## Context
- `Step4Report.vue` usa layout split: painel esquerdo com conteudo do relatorio + painel direito com workflow timeline (agent logs, tool calls, llm responses, metricas) + console logs inferior.
- `ReportOutline.vue` ja existe como sidebar sticky com navegacao por headings (extraidos do DOM), mas so aparece em `ReportView.vue` (rota `/report/:reportId`), nao dentro do Step4 no workbench.
- O relatorio ja tem secoes colapsaveis (`collapsedSections`), badges de due diligence (`dd-tese`, `dd-evidencias`, etc.) e estado de carregamento por secao.
- Nao existe toggle de modo foco no estado atual.

## Task 4: Indice navegavel por secoes dentro de Step4Report

<action>
Em `frontend/src/components/Step4Report.vue`:
1. Renderizar um mini-indice de secoes no topo do painel esquerdo, condicional ao modo foco (`v-if="focusMode"`), usando as secoes de `reportOutline.sections`.
2. O indice deve ser um `<nav class="report-section-nav">` com lista de links.
3. Cada link deve mostrar o numero da secao + titulo e, ao clicar, scrollar suavemente ate a secao correspondente usando `scrollIntoView({ behavior: 'smooth', block: 'start' })`.
4. O indice deve destacar a secao atualmente visivel usando IntersectionObserver (similar ao `ReportOutline.vue`), com classe `active` no item correspondente.
5. O indice deve ser `position: sticky; top: 0` dentro do painel esquerdo para permanecer visivel durante scroll.
</action>
