# Plan: Deduplication and Rebuild Skip

## Goal
Prevent duplicate and unchanged content from being sent to Zep during rebuild workflows.

## Must-Haves
1. Exact duplicate chunks are removed before `add_batch` ingestion.
2. Rebuild requests compare current extracted text and ontology against stored signatures.
3. Unchanged builds skip unnecessary Zep ingestion instead of resending all chunks.
4. Skip behavior is visible through logs or task messages.
5. Forced rebuild paths still work when users explicitly need a full resend.

<task>
  <id>2</id>
  <name>Wire deduplication and skip logic into build flow</name>
  <files>
    - backend/app/services/graph_builder.py
    - backend/app/api/graph.py
  </files>
  <read_first>
    - backend/app/services/graph_builder.py
    - backend/app/api/graph.py
    - backend/app/models/project.py
  </read_first>
  <action>
    1. In `GraphBuilderService.build_graph_async`, compute the text and ontology signatures before starting the worker.
    2. In the worker (`_build_graph_worker`), after splitting text into chunks, call `TextProcessor.deduplicate_chunks` before `add_text_batches`.
    3. In `backend/app/api/graph.py` build endpoint, before creating a task:
       - Compute `text_signature` from `ProjectManager.get_extracted_text(project_id)`
       - Compute `ontology_signature` from `json.dumps(project.ontology, sort_keys=True)`
       - If both signatures match the stored values AND `force` is `False`, skip the build:
         - Return success with a message indicating no changes detected
         - Do NOT create a TaskManager task
         - Do NOT call Zep
       - If signatures differ OR `force` is `True`, store the new signatures in the project and proceed with the normal build.
    4. Update task messages so that a skipped build logs "No changes detected — skipping rebuild" and a normal build logs "Building graph..." as usual.
  </action>
  <verify>
    <automated>cd backend && uv run pytest -q tests/services/test_graph_rebuild_skip.py</automated>
  </verify>
  <acceptance_criteria>
    - Rebuild with unchanged text and ontology skips Zep ingestion.
    - Skip response is immediate (no task created).
    - Skip behavior is visible in logs or response messages.
    - Force rebuild bypasses signature comparison.
    - Deduplication reduces duplicate chunks before add_batch.
  </acceptance_criteria>
</task>
