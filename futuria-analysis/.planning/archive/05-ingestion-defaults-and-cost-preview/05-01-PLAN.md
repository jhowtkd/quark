# Plan: Ingestion Defaults and Cost Preview

## Goal
Tune the graph-build defaults to reduce Zep spend, add a backend preflight cost estimate, and force the user to see that estimate before the real ingestion task starts.

## Must-Haves
1. Default chunk size and overlap move to 300/30 for new projects, and legacy persisted 500/50 records are normalized to the same baseline while still honoring manual overrides.
2. Preview reports chunk count, total bytes, and estimated credits from the saved extracted text before any task creation.
3. High-cost builds render a warning before the real build begins.
4. Confirmed builds still create the existing task, poll status, refresh the graph, and leave ontology/schema behavior untouched.

## Threat Model

| Boundary | Description |
|----------|-------------|
| Browser -> `/api/graph/build` | Untrusted chunk overrides, preview flags, and confirm actions cross into the backend. |
| Backend -> Zep | Only the confirmed build may cross into the billed ingestion boundary. |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering / DoS | Chunk settings parser in `graph.py` | mitigate | Clamp `chunk_size` / `chunk_overlap` server-side, reuse the normalized values for both preview and build, and reject negative or inverted ranges before any split is attempted. |
| T-05-02 | Tampering | Preview/build handoff in `MainView.vue` | mitigate | Clear stale preview state whenever the user edits settings and require the preview branch to succeed again before the confirm action can start the task. |
| T-05-03 | DoS | Build route and task lifecycle | mitigate | Keep `preview` before `TaskManager().create_task(...)` and before Zep writes so the confirmation gate cannot accidentally spend credits or create a misleading task. |
