# Plan: Add Edge Relation Translation to GraphPanel and Locale Files

## Goal
Translate English edge/relation labels to Portuguese in the graph UI using vue-i18n, without modifying backend data or simulation flows.

## Must-Haves (Goal-Backward Verification)

1. `locales/pt.json` contains a `graph.relations` object mapping all required edge types to Portuguese.
2. `GraphPanel.vue` imports the i18n instance and exposes a `getTranslatedEdgeLabel(name)` helper.
3. D3 link labels (`.text(...)`) pass edge names through `getTranslatedEdgeLabel` before rendering.
4. The edge detail panel and self-loop headers pass displayed relation names through `getTranslatedEdgeLabel`.
5. Unmapped edge types fall back to the original English text via `i18n.global.te(key)` guard.
6. `npm run build` in `frontend/` completes with exit code 0.

<task id="2">
  <title>Import i18n and create getTranslatedEdgeLabel helper in GraphPanel.vue</title>
  <read_first>
    - frontend/src/components/GraphPanel.vue
    - frontend/src/i18n/index.js
  </read_first>
  <action>
    In `frontend/src/components/GraphPanel.vue`, inside the `<script setup>` block:
    1. Add the import statement immediately after the existing `import * as d3 from 'd3'` line:
       ```javascript
       import i18n from '../i18n'
       ```
    2. Add the following helper function before the `renderGraph` function definition:
       ```javascript
       const getTranslatedEdgeLabel = (name) => {
         if (!name) return name
         const key = `graph.relations.${name}`
         return i18n.global.te(key) ? i18n.global.t(key) : name
       }
       ```
  </action>
  <acceptance_criteria>
    - `grep 'import i18n' frontend/src/components/GraphPanel.vue` returns a match.
    - `grep 'getTranslatedEdgeLabel' frontend/src/components/GraphPanel.vue` returns at least one match.
    - `grep 'i18n.global.te' frontend/src/components/GraphPanel.vue` returns a match.
  </acceptance_criteria>
</task>
